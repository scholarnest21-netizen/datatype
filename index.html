Chapter 01

Python Interpreter & Interactive Mode

Python Interpreter is a program that reads Python source code and executes it line by line. Unlike compiled languages (C/C++), Python translates and runs code on the fly without a separate compilation step.

Python has two primary modes of operation:

  • Script Mode — you write code in a .py file and run it: python myfile.py
  • Interactive Mode (REPL) — you type code directly into the terminal and see results instantly. REPL stands for Read – Evaluate – Print – Loop.

Interactive mode is ideal for experimenting, testing snippets, and learning. You launch it by typing python or python3 in your terminal.

Chapter 02

Debugging

Debugging is the process of finding and fixing errors (bugs) in a program. Python has three types of errors:

  • Syntax Errors — code violates Python's grammar rules; the interpreter refuses to run it.
  • Runtime Errors — code is grammatically correct but fails during execution (e.g. dividing by zero).
  • Semantic / Logic Errors — code runs without crashing but produces wrong results.

Common debugging strategies: read the error message carefully, use print() statements to trace values, isolate the problem to the smallest piece of code, and use Python's built-in debugger (pdb).

Chapter 03

Values & Types

Every value in Python has a data type that determines what kind of data it holds and what operations can be performed on it. Python is dynamically typed — you don't declare types; Python figures them out automatically.

Use the built-in type() function to check any value's type. The five core types are:

int

Integer

Whole numbers, positive or negative, with no decimal point. e.g. 5, -99, 1000

float

Float

Numbers with a decimal point or in scientific notation. e.g. 3.14, -0.5, 2e8

bool

Boolean

Logical values: only True or False. Used in conditions and comparisons.

str

String

A sequence of characters enclosed in quotes. e.g. "hello", 'Python'

list

List

An ordered, mutable collection of items in square brackets. e.g. [1, 2, 3]

Chapter 04

Variables

A variable is a named container that stores a value in memory. In Python, you create a variable simply by assigning a value to a name using the = operator. There is no need to declare the type first.

Python variable naming rules:

  • Must start with a letter (a–z, A–Z) or underscore (_)
  • Can contain letters, digits, and underscores
  • Case-sensitive: age ≠ Age ≠ AGE
  • Cannot use Python keywords (if, for, True, etc.)
  • Convention: use snake_case for multi-word names

Python is dynamically typed: a variable can hold any type and can change type.

Chapter 05

Expressions

An expression is any combination of values, variables, and operators that Python can evaluate to produce a result. Expressions can be as simple as a single number (42) or as complex as a multi-step formula.

Types of expressions in Python:

  • Arithmetic Expressions — use math operators: + - * / // % **
  • Comparison Expressions — compare two values: == != < > <= >=
  • Logical Expressions — combine boolean values: and or not
  • String Expressions — concatenation, repetition

Chapter 06

Statements

A statement is a complete instruction that Python can execute. Unlike an expression (which produces a value), a statement performs an action. Common types include:

  • Assignment statement — stores a value in a variable
  • Print statement — outputs data to the console
  • Conditional statement — makes decisions (if / elif / else)
  • Augmented assignment — shorthand for updating a variable
  • Import statement — loads external modules

Each logical line in Python is typically one statement. Use a semicolon (;) to put multiple statements on one line (uncommon), or backslash (\) to continue a statement across lines.

Assignment Statements
# Basic assignment
price = 100

# Augmented assignment operators

price -= 20     # price = price - 20  → 130
price *= 2      # price = price * 2   → 260
price //= 3     # price = price // 3  → 86
print(price)    # 86

# Multiple statements on one line (not recommended)
a = 1; b = 2; print(a + b)   # 3

# Line continuation
total = 100 + 200 \
      + 300 + 400
print(total)    # 1000

Chapter 07

Tuple Assignment

A tuple is an ordered, immutable sequence of values enclosed in parentheses (or even without them). Unlike lists, tuples cannot be modified after creation.

Tuple assignment (also called unpacking) lets you assign multiple variables from a tuple in a single statement. Python matches positions: the first value goes to the first variable, and so on. The number of variables must exactly match the number of values (unless you use *).

This is one of Python's most elegant features, widely used for returning multiple values from functions, swapping variables, and iterating over paired data.

tuple

Immutable

Cannot be changed after creation. Elements cannot be added, removed, or modified.

list

Mutable

Can be changed. Elements can be added, removed, or modified at any time.

Chapter 08

Precedence of Operators

Operator precedence determines the order in which operators are evaluated in an expression with multiple operators. Python follows PEMDAS / BODMAS — but with Python-specific additions.

Higher precedence operators are evaluated first. When operators have the same precedence, associativity decides the order — most operators are left-to-right (left associative), except exponentiation (**) which is right-to-left.

Always use parentheses to make complex expressions clearer and avoid bugs.

Chapter 09

Comments

A comment is text in source code that Python completely ignores during execution. Comments exist solely for human readers — they explain what the code does, why a decision was made, or provide important notes.

Python supports two styles of comments:

  • Single-line comment — starts with #; everything after # on that line is a comment
  • Multi-line comment (docstring) — triple quotes """...""" or '''...''' spanning multiple lines. When placed at the start of a function/class/module, Python treats it as a documentation string (docstring).

Good commenting practices: explain why, not what; keep comments up to date; avoid obvious comments; write self-documenting code first.

Single-Line Comments
# This is a full-line comment
radius = 7     # inline comment — after the code
pi     = 3.14159

# Temporarily disable code during testing
area = pi * radius ** 2   # πr²
# circumference = 2 * pi * radius   ← disabled
print(f"Area = {area:.2f}")
Area = 153.94
